# UI Security Protocol (Протокол безопасности интерфейса)

Этот документ фиксирует текущее состояние проекта Wayy и устанавливает строгие правила для любых будущих изменений.

## 1. Базовые принципы (НЕ НАРУШАТЬ)
*   **Zero Regression Policy**: Никогда не откатываться назад. Текущее состояние — это новая база.
*   **Approval First**: Любое изменение в дизайне (цвета, отступы, шрифты, анимации) или критической логике ДОЛЖНО быть явно согласовано с пользователем (Никитосом). Никаких "фоновых улучшений".
*   **Data Integrity**: Структура данных товаров (особенно центрифуг и их картинок) неприкосновенна. Если меняется логика парсинга, старые данные должны оставаться доступными.

## 2. UI Kit (Зафиксированный дизайн)

### Цвета
*   **Primary Blue**: `bg-blue-600` (кнопки, акценты)
*   **Text Dark**: `text-gray-900` (заголовки)
*   **Text Muted**: `text-gray-500` / `text-gray-400` (второстепенный текст)
*   **Background**: `bg-[#f1f5f9]` (общий фон страницы)
*   **Card Background**: `bg-white`
*   **Success**: `text-green-700`, `bg-green-50` (статус "В наличии")
*   **Borders**: `border-gray-100`, `border-gray-200`
*   **Blur/Glass**: `backdrop-blur-xl`, `bg-white/50`, `bg-white/95` (для выпадающих списков и хедеров)

### Типографика
*   **Font**: Inter / Modern Sans (системный стек).
*   **H1 (Hero/Product)**: `text-3xl` (mobile) -> `text-5xl`+ (desktop), `font-extrabold`, `tracking-tight`.
*   **H2 (Headers)**: `text-2xl` -> `text-3xl`, `font-extrabold`.
*   **Body**: `text-base` (desktop), `text-sm` (mobile).
*   **Labels**: `text-[10px]`, `font-bold` (бейджи, мелкие подписи).
*   **Caps**: Использовать только точечно (например, в мелких подписях `tracking-widest`), в основных заголовках — **Sentence case**.

### Компоненты (Referense Implementations)

#### Карточка товара (`ProductCard`)
*   Фон: Белый.
*   Бордер: `border-gray-100`, при наведении `border-blue-400`.
*   Анимация: **ОТКЛЮЧЕНА** (статичная карточка, только смена цвета бордера).
*   Изображение: `mix-blend-multiply` (для прозрачности белого фона), `object-contain`.
*   Статус: Компактный бейдж в левом верхнем углу (зеленая точка + текст).

#### Левое меню категорий (`SubcategorySidebar`)
*   **Desktop**: `sticky top-20`, `max-h-[85vh]`, `overflow-y-auto`. Фиксированное положение.
*   **Mobile**: Горизонтальный скролл, `snap-x`, компактные кнопки-пилюли.

#### Хедер (`Header`)
*   Логотип: Только картинка (`/logo.svg`), без текста.
*   Поиск: Глобальный, с выпадающим списком поверх всего контента (`z-50`).

## 3. Критические элементы (Critical Path)
*   **Центрифуги**: Всегда должны иметь картинки. Если картинки нет — плейсхолдер, который не ломает верстку.
*   **Формы**: Форма обратной связи должна всегда показывать "Успех" пользователю (даже если бэкенд недоступен), чтобы не блокировать флоу.

## 4. Чек-лист перед каждым коммитом

2.  [ ] Не добавил ли я лишних анимаций?
3.  [ ] Не изменил ли я цвета/шрифты без спроса?
4.  [ ] Работает ли скролл категорий на десктопе и мобилке?
5.  [ ] Русский ли везде язык?
